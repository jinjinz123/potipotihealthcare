# Firestore Security Specification

This document details the Zero-Trust security rules schema and data invariants for the Sleep & Well-being tracker applet.

## 1. Core Data Invariants

1. **Owner-Isolations (Master Gate)**: A user can only access, view, or write their own documents. All sub-resources (like `records`) under `/users/{userId}` inherit permission from the parent `{userId}` match.
2. **Identity Integrity**: No user may read, write, or alter another user's profile settings, lists, or sleep tracking sheets.
3. **PWA Offline Sync Robustness**: Real-time batch logs and offline syncing are completely owner-gated.
4. **Hour Representation Validation**: The profile option `hourRep` must strictly be either `"1-24"` or `"0-23"`.
5. **No Blind Global Reads**: Global listing of users or sleep record logs is strictly disabled.
6. **Immutable Creation Timestamps**: Fields like `createdAt` must match `request.time` exactly at creation and are completely immutable during updates.
7. **Strict Update Validation**: During updates, only the `updatedAt` and specific fields can be modified.

## 2. The Dirty Dozen Attack Payloads

The following payloads attempt to violate security boundaries and must return `PERMISSION_DENIED`:

### Payload 1: Profile Theft (Spoofing UID)
- **Target Path**: `/users/victim_user_uid`
- **Requester UID**: `attacker_uid`
- **Attempt**: Write a profile under victim's path.
- **Fail Criteria**: Allowing writes where `request.auth.uid != userId`.

### Payload 2: Self-Promotion (Forging ID Verification)
- **Target Path**: `/users/attacker_uid`
- **Attempt**: Create a profile without establishing email verification.
- **Fail Criteria**: Bypassing verification state checks.

### Payload 3: Invalid Hour Format Poisoning
- **Target Path**: `/users/user_uid`
- **Attempt**: Set `hourRep` to `"invalid_hour_representation_string"`.
- **Fail Criteria**: Allowing free-form strings for structured setting enums.

### Payload 4: Arbitrary User Record Scraping
- **Target Path**: `/users/victim_user_uid/records/2026-06-18`
- **Requester UID**: `attacker_uid`
- **Attempt**: Perform `get` or `list` query to peek at victim's records.
- **Fail Criteria**: Allowing reading records of another user.

### Payload 5: Date Poisoning Attack
- **Target Path**: `/users/user_uid/records/invalid-date-format-leak`
- **Attempt**: Write daily record document with custom garbage document id.
- **Fail Criteria**: Bypassing date-matching format guards.

### Payload 6: Mega-Memo Allocation (Denial of Wallet)
- **Target Path**: `/users/user_uid/records/2026-06-18`
- **Attempt**: Insert 10MB of garbage text into the `memo` field of a sleep record to inflate storage costs.
- **Fail Criteria**: Allowing extremely large memo writes (must be ≤ 10,000 characters).

### Payload 7: Ghost Field Pollution
- **Target Path**: `/users/user_uid/records/2026-06-18`
- **Attempt**: inject `ghost_key: "malicious_injected_payload"` field during record sync.
- **Fail Criteria**: Permitting keys outside of the designated schema (`slots`, `memo`, `dateStr`, `updatedAt`, `createdAt`).

### Payload 8: Immutable Backdating
- **Target Path**: `/users/user_uid`
- **Attempt**: Overwrite `createdAt` with a timestamp 10 years in the past.
- **Fail Criteria**: Allowing the immutable `createdAt` field to shift.

### Payload 9: Future Timestamp Forgery
- **Target Path**: `/users/user_uid/records/2026-06-18`
- **Attempt**: Set `updatedAt` to a future timestamp (e.g. 50 years on) generated by a client's system clock.
- **Fail Criteria**: Bypassing `request.time` enforcement.

### Payload 10: Invalid Sleep Symbols Injection
- **Target Path**: `/users/user_uid/records/2026-06-18`
- **Attempt**: Register a slot status using an unrecognized character count/symbols like `"A"`, `"D"`, `"awake"`.
- **Fail Criteria**: Accepting status symbols other than `'★'`, `'S'`, `'×'`, `'○'`, `'－'` and empty.

### Payload 11: Non-existent Owner Record Creation
- **Target Path**: `/users/ghost_user_uid/records/2026-06-18`
- **Attempt**: Create a record slot list under a non-existent parent user profile.
- **Fail Criteria**: Allowing orphans before the root profile is established.

### Payload 12: Administrative Escalation Bypass
- **Target Path**: `/users/user_uid`
- **Attempt**: Forging admin claims inside document profiles.
- **Fail Criteria**: Self-assigning roles or permissions through simple user-profile fields.
